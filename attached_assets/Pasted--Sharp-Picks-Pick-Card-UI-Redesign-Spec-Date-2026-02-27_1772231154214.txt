# Sharp Picks — Pick Card UI Redesign Spec

**Date:** 2026-02-27  
**Component:** Pick History / Feed Card List  
**Priority:** High

-----

## Overview

The current pick card list communicates the right data but lacks visual hierarchy and scannability. These changes improve at-a-glance readability without a full redesign — same dark theme, same data, sharper execution.

-----

## 1. Left Border Accent (Outcome Indicator)

Add a 4px left border to each card row that communicates result at a glance before the user reads anything.

```css
.pick-card {
  border-left: 4px solid transparent;
  padding-left: 14px;
}

.pick-card.win    { border-left-color: #22c55e; } /* green */
.pick-card.loss   { border-left-color: #ef4444; } /* red */
.pick-card.pending { border-left-color: #6b7280; } /* gray */
.pick-card.withdrawn { border-left-color: #7c3aed; } /* purple */
```

**Why:** Users can scan an entire feed vertically by color before reading a single word. This is the highest-ROI change on the list.

-----

## 2. Status Badge → Pill Component

Replace plain colored text (e.g. “Pending”, “Withdrawn”) with pill-shaped badges that have a tinted background matching the status color.

```css
.status-badge {
  display: inline-flex;
  align-items: center;
  padding: 3px 10px;
  border-radius: 999px;
  font-size: 11px;
  font-weight: 600;
  letter-spacing: 0.04em;
  text-transform: uppercase;
}

.status-badge.pending    { background: rgba(107,114,128,0.15); color: #9ca3af; }
.status-badge.win        { background: rgba(34,197,94,0.12);   color: #22c55e; }
.status-badge.loss       { background: rgba(239,68,68,0.12);   color: #ef4444; }
.status-badge.withdrawn  { background: rgba(124,58,237,0.12);  color: #a78bfa; }
```

**Why:** Plain colored text is easy to miss. A contained shape with a tinted fill registers as a distinct UI element, not just a label.

-----

## 3. Typography Hierarchy — Three Tiers

Current: pick name, matchup, and date are visually similar weight.  
Target: three clearly distinct tiers.

```css
/* Tier 1 — Pick name (e.g. "Milwaukee Bucks +8.5") */
.pick-title {
  font-size: 16px;
  font-weight: 700;
  color: #f9fafb;
  line-height: 1.3;
}

/* Tier 2 — Matchup (e.g. "New York Knicks @ Milwaukee Bucks") */
.pick-matchup {
  font-size: 13px;
  font-weight: 400;
  color: #9ca3af;
  margin-top: 2px;
}

/* Tier 3 — Date */
.pick-date {
  font-size: 11px;
  font-weight: 400;
  color: #6b7280;
  margin-top: 2px;
}
```

**Why:** Right now the eye has nowhere to land first. This gives the pick name clear dominance and pushes supporting info into the background.

-----

## 4. Result & Edge Layout — Separate Roles

The result value (`+1.02u`) and edge percentage (`7.9% edge`) should not compete visually. Result is the outcome; edge is context.

```css
/* Result value — large, dominant */
.pick-result {
  font-size: 17px;
  font-weight: 700;
  line-height: 1;
}
.pick-result.positive { color: #22c55e; }
.pick-result.negative { color: #ef4444; }

/* Edge percentage — small, muted, below result */
.pick-edge {
  font-size: 11px;
  font-weight: 500;
  color: #6b7280;
  margin-top: 3px;
  text-align: right;
}
```

**Layout change:** Stack these vertically (result on top, edge below) instead of edge appearing to float independently.

-----

## 5. Edge Confidence Indicator (Optional Enhancement)

Instead of displaying edge as plain text, add a small visual indicator using 4 dots scaled to confidence tier.

|Edge Range|Dots Filled|
|----------|-----------|
|3.5–5.9%  |● ○ ○ ○    |
|6–7.9%    |● ● ○ ○    |
|8–9.9%    |● ● ● ○    |
|10%+      |● ● ● ●    |

```jsx
// React example
const EdgeDots = ({ edge }) => {
  const filled = edge >= 10 ? 4 : edge >= 8 ? 3 : edge >= 6 ? 2 : 1;
  return (
    <div className="edge-dots">
      {[1,2,3,4].map(i => (
        <span key={i} className={i <= filled ? 'dot filled' : 'dot'} />
      ))}
    </div>
  );
};
```

```css
.edge-dots { display: flex; gap: 3px; justify-content: flex-end; }
.dot { width: 5px; height: 5px; border-radius: 50%; background: #374151; }
.dot.filled { background: #22c55e; }
```

**Why:** Makes high-edge picks visually distinct from minimum-threshold picks without adding any numeric noise.

-----

## 6. Card Spacing & Dividers

Increase vertical breathing room per card. Current padding feels compressed on dark backgrounds.

```css
.pick-card {
  padding: 16px 16px 16px 18px; /* extra left for border accent */
  min-height: 72px;
}

/* Replace hard dividers with subtle spacing */
.pick-card + .pick-card {
  border-top: 1px solid rgba(255,255,255,0.06);
}
```

**Why:** Dark UIs need more padding than light ones — high contrast makes density feel claustrophobic faster.

-----

## 7. Chevron Alignment

The `>` arrow on the right is currently vertically misaligned with the result value. It should be vertically centered relative to the entire card height, not just the result text.

```css
.pick-card-right {
  display: flex;
  flex-direction: column;
  align-items: flex-end;
  justify-content: center; /* vertically center within card */
  gap: 4px;
}

.pick-chevron {
  color: #4b5563;
  font-size: 14px;
  margin-top: 4px; /* subtle offset below result+edge stack */
}
```
